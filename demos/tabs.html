Programmatic Control

Control the active tab via the value property.

First Second Third
First tab content Second tab content Third tab content 
const tabs = document.getElementById('my-tabs')

// Get current value
console.log(tabs.value) // "first"

// Set programmatically
tabs.value = 'second'

// Listen for changes
tabs.addEventListener('change', (e) => {
  console.log('Selected:', e.detail.value)
})

Styling with Tachyons

Tabs are unstyled by default. Use Tachyons or custom CSS for appearance. The [aria-selected="true"] selector targets the active tab.

Dashboard Analytics Reports
Dashboard overview with key metrics. Analytics and user behavior insights. Generated reports and exports. 
/* Target active tab with attribute selector */
ytz-tab[aria-selected="true"] {
  border-bottom-color: blue;
  font-weight: 600;
}

/* Hover state */
ytz-tab:hover {
  background: rgba(0,0,0,0.05);
}

/* Focus state for accessibility */
ytz-tab:focus {
  outline: 2px solid blue;
  outline-offset: -2px;
}

Keyboard Navigation

Tabs implement the WAI-ARIA tabs pattern with roving tabindex. Only the active tab is in the tab order; arrow keys move between tabs.

Key Action
Tab Move focus to active tab, then to panel content
← / → Navigate between tabs (horizontal orientation)
↑ / ↓ Navigate between tabs (vertical orientation)
Home Move to first tab
End Move to last tab

Try it: focus a tab below and use arrow keys to navigate.

Tab 1 Tab 2 Tab 3 Tab 4
Use arrow keys to navigate between tabs. Press Home to jump to the first tab. Press End to jump to the last tab. Navigation wraps from last to first and vice versa.

API Reference

ytz-tabs Attributes

Attribute Type Description
value string ID of the currently selected panel
orientation "horizontal" | "vertical" Arrow key direction (default: horizontal)

ytz-tab Attributes

Attribute Type Description
panel string ID of the associated tabpanel (required)

ytz-tabpanel Attributes

Attribute Type Description
id string Panel identifier, must match a tab's panel attribute

Events

Event Target Detail Description
change ytz-tabs { value: string } Fired when active tab changes

ARIA Attributes (Auto-managed)

Element ARIA
ytz-tabs role="tablist"
ytz-tab role="tab", aria-selected, aria-controls, tabindex
ytz-tabpanel role="tabpanel", aria-labelledby

React Usage

import { useState } from 'react'
import { Tabs, Tab, TabPanel } from '@yetzirah/react'

function Settings() {
  const [activeTab, setActiveTab] = useState('account')

  return (
    <Tabs value={activeTab} onChange={setActiveTab}>
      <div className="flex">
        <Tab panel="account">Account</Tab>
        <Tab panel="password">Password</Tab>
        <Tab panel="notifications">Notifications</Tab>
      </div>
      <TabPanel id="account">Account settings...</TabPanel>
      <TabPanel id="password">Password settings...</TabPanel>
      <TabPanel id="notifications">Notification settings...</TabPanel>
    </Tabs>
  )
}

  ...

Migrating from MUI Tabs

MUI

<Tabs value={value} onChange={handleChange}>
  <Tab label="Item One" />
  <Tab label="Item Two" />
</Tabs>
<TabPanel value={value} index={0}>
  Content One
</TabPanel>
<TabPanel value={value} index={1}>
  Content Two
</TabPanel>

Yetzirah

<Tabs value={value} onChange={setValue}>
  <Tab panel="one">Item One</Tab>
  <Tab panel="two">Item Two</Tab>
  <TabPanel id="one">
    Content One
  </TabPanel>
  <TabPanel id="two">
    Content Two
  </TabPanel>
</Tabs>

Key Differences

MUI Yetzirah
value={0} (index) value="panel-id" (string ID)
onChange={(e, val) => ...} onChange={val => ...}
<Tab label="..." /> <Tab panel="id">...</Tab>
<TabPanel value={v} index={0}> <TabPanel id="panel-id">
variant="scrollable" Use CSS overflow-x: auto
orientation="vertical" orientation="vertical"

Key difference: Yetzirah uses string IDs to link tabs and panels, making the relationship explicit. Panels are children of the tabs container, not separate components.
