Tooltip

Positioned hint text on hover/focus with configurable placement and delay.

Basic Usage

Wrap any element with <ytz-tooltip>. The first child is the trigger, and the [slot="content"] element contains the tooltip text.

This is a helpful tooltip! 
<ytz-tooltip>
  <button class="ph3 pv2 br2 bg-blue white bn pointer">
    Hover me
  </button>
  <span slot="content">This is a helpful tooltip!</span>
</ytz-tooltip>

Placements

Use the placement attribute to position the tooltip. It will automatically flip if there's not enough space.

Tooltip on top Tooltip on bottom Tooltip on left Tooltip on right 
<ytz-tooltip placement="top">...</ytz-tooltip>
<ytz-tooltip placement="bottom">...</ytz-tooltip>
<ytz-tooltip placement="left">...</ytz-tooltip>
<ytz-tooltip placement="right">...</ytz-tooltip>

Show Delay

Use the delay attribute to add a delay before the tooltip appears on hover. This prevents tooltips from appearing when quickly moving the mouse across the page.

Appears immediately Waited 300ms to show Waited 500ms to show 
<ytz-tooltip delay="500">
  <button>Hover me</button>
  <span slot="content">Delayed tooltip</span>
</ytz-tooltip>

Custom Offset

The offset attribute controls the gap between the trigger and tooltip. Default is 8px.

Close to trigger Default distance More space from trigger 
<ytz-tooltip offset="16">
  <button>Hover me</button>
  <span slot="content">Farther away</span>
</ytz-tooltip>

Focus Support

Tooltips show on focus for keyboard accessibility. Unlike hover, focus shows the tooltip immediately without delay.

Tab to the input to see the tooltip on focus:

Tooltips show on focus too! No delay for accessibility. 
<ytz-tooltip>
  <input type="text" placeholder="Focus me">
  <span slot="content">Keyboard accessible!</span>
</ytz-tooltip>

Auto-flip

Tooltips automatically flip when there's not enough space in the preferred direction. The data-placement attribute on the tooltip content reflects the actual placement.

I flip to the left when there's no room! 
/* CSS can target actual placement */
[data-placement="left"]::after { /* arrow pointing right */ }
[data-placement="right"]::after { /* arrow pointing left */ }

Rich Content

Tooltip content can contain HTML. Users can hover over the tooltip content without it disappearing.

Pro Tip You can hover over this tooltip content without it disappearing! 
<ytz-tooltip>
  <button>Hover for details</button>
  <span slot="content">
    <strong>Title</strong><br>
    <span class="f7">Detailed description</span>
  </span>
</ytz-tooltip>

Styling

Yetzirah provides no default styles. Add your own with CSS targeting [role="tooltip"]. 

/* Basic dark tooltip */
[role="tooltip"] {
  background: #333;
  color: white;
  padding: 0.375rem 0.625rem;
  border-radius: 4px;
  font-size: 0.875rem;
  z-index: 1000;
  max-width: 250px;
}

/* Arrow based on actual placement */
[role="tooltip"][data-placement="top"]::after {
  content: '';
  position: absolute;
  top: 100%;
  left: 50%;
  margin-left: -5px;
  border: 5px solid transparent;
  border-top-color: #333;
}

[role="tooltip"][data-placement="bottom"]::after {
  content: '';
  position: absolute;
  bottom: 100%;
  left: 50%;
  margin-left: -5px;
  border: 5px solid transparent;
  border-bottom-color: #333;
}

API Reference

Attributes

Attribute Type Default Description
placement String "top" Position: "top", "bottom", "left", "right"
delay Number 0 Show delay in milliseconds (hover only)
offset Number 8 Gap between trigger and tooltip in pixels

Events

Event Description
show Fired when tooltip becomes visible
hide Fired when tooltip is hidden

ARIA

  • Trigger gets aria-describedby pointing to tooltip
  • Tooltip content gets role="tooltip"
  • Content gets data-placement with actual placement

JavaScript API

const tooltip = document.querySelector('ytz-tooltip')

// Read properties
console.log(tooltip.placement) // "top"
console.log(tooltip.delay)     // 0
console.log(tooltip.offset)    // 8

// Set properties
tooltip.placement = 'bottom'
tooltip.delay = 300

// Show/hide programmatically
tooltip.show()
tooltip.hide()

// Listen for events
tooltip.addEventListener('show', () => console.log('Tooltip shown'))
tooltip.addEventListener('hide', () => console.log('Tooltip hidden'))

React Usage

import { Tooltip, Button } from '@grimoire/yetzirah-react'

function App() {
  return (
    <Tooltip content="Helpful hint" placement="top">
      <Button>Hover me</Button>
    </Tooltip>
  )
}

// With callbacks
<Tooltip
  content="Tracked tooltip"
  placement="bottom"
  delay={300}
  onShow={() => analytics.track('tooltip_shown')}
  onHide={() => analytics.track('tooltip_hidden')}
>
  <button>Hover me</button>
</Tooltip>

// With className for tooltip styling
<Tooltip content="Styled tooltip" className="custom-tooltip">
  <button>Hover me</button>
</Tooltip>
